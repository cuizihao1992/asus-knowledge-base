# 资料价值评估与整理规则

生成日期：2026-06-19

目标：在真正整理 D/E 盘资料前，先统一“怎么评价价值、怎么分优先级、怎么处理文件”。本规则只用于生成整理建议，不代表自动移动或删除文件。

## 总原则

1. 不自动删除任何文件。
2. 不自动移动任何原始资料。
3. 不自动解压大压缩包。
4. 不上传模型、视频、数据库、安装包和原始项目资料。
5. 任何删除、移动、压缩、归档操作，都必须先生成清单，再由人工确认。
6. 默认操作是“索引”和“摘要”，不是“清理磁盘”。

## 资料分类

| 分类 | 含义 | 处理方式 |
| --- | --- | --- |
| 核心保留 | 明确有项目、经验、领域或复用价值 | 建立索引，必要时做摘要 |
| 优先摘要 | 文档或项目内容值得沉淀成知识 | 生成 Markdown 摘要 |
| 资产索引 | 模型、视频、GIS 数据、大文件 | 只记录路径、体量、格式和关联关系 |
| 归档保留 | 有历史价值，但短期不再处理 | 记录后归档，不深入分析 |
| 待确认 | 名称、来源或用途不清楚 | 暂不处理，列入确认清单 |
| 可删除候选 | 可能重复、过期、缓存或安装包 | 只列为候选，不自动删除 |
| 禁止处理 | 系统目录、权限敏感目录、运行环境关键目录 | 不扫描或只浅层记录 |

## 优先级

| 优先级 | 标准 | 示例 |
| --- | --- | --- |
| P0 | 关系到当前知识库、项目经验或高价值业务资料 | `D:\zhc`、`E:\SUTPC`、`E:\Data` |
| P1 | 可形成专题摘要或项目卡片 | mini-transformer、quant-system、BIM 集成方案 |
| P2 | 可作为资料索引，但暂不摘要 | 学习资料、Cesium 示例、SuperMap 资料 |
| P3 | 只需归档或待确认 | 下载池、大压缩包、老安装包 |
| P4 | 明显低价值或可删除候选 | 缓存、重复安装包、临时文件 |

## 价值评分

每个资料可以按 5 个维度打分，每项 0 到 3 分。

| 维度 | 0 分 | 1 分 | 2 分 | 3 分 |
| --- | --- | --- | --- | --- |
| 项目关联 | 无项目 | 可能有关 | 明确属于项目 | 核心项目资料 |
| 复用价值 | 无复用 | 偶尔参考 | 可复用方法/代码 | 可直接迁移能力 |
| 领域价值 | 无领域信息 | 一般资料 | BIM/GIS/AI/量化等专业资料 | 稀缺领域经验 |
| 可理解性 | 无法判断 | 需打开确认 | 文件名和结构清楚 | 已有 README/文档 |
| 风险价值 | 可轻易再下载 | 丢失影响小 | 丢失会增加工作量 | 丢失会破坏项目脉络 |

建议判定：

| 总分 | 建议 |
| ---: | --- |
| 12-15 | 核心保留，优先摘要 |
| 8-11 | 保留索引，择机摘要 |
| 4-7 | 待确认或归档保留 |
| 0-3 | 可删除候选，但仍需确认 |

## 文件类型处理规则

| 类型 | 默认分类 | 处理方式 |
| --- | --- | --- |
| `.md`、`README` | 优先摘要 | 读取并总结 |
| `.doc`、`.docx` | 优先摘要/待确认 | 先索引，再挑选摘要 |
| `.pdf` | 优先摘要/资产索引 | 先记录标题和路径 |
| `.ppt`、`.pptx` | 优先摘要 | 适合提炼项目背景和方案 |
| `.xls`、`.xlsx` | 优先摘要/数据索引 | 记录用途，谨慎读取 |
| `.py`、`.js`、`.ts` | 项目索引 | 不逐行分析，先看入口和结构 |
| `.fbx`、`.obj`、`.b3dm`、`.cmpt` | 资产索引 | 不上传，不打开大文件，记录路径 |
| `.mp4`、`.avi`、`.hav` | 资产索引 | 只记录，不分析内容 |
| `.zip`、`.rar`、`.7z` | 待确认 | 不自动解压 |
| `.exe`、`.msi` | 可删除候选/工具索引 | 只记录，不运行 |
| `.db`、`.sqlite` | 资产索引 | 不上传，必要时只记录表用途 |
| 缓存/构建产物 | 可删除候选 | 只列清单，确认后处理 |

## 可删除候选判定

注意：可删除候选不等于删除。

满足以下条件之一，可以列入“可删除候选”：

- 明确是缓存目录。
- 明确是构建产物。
- 明确是重复安装包。
- 明确是临时下载文件。
- 已有更完整版本，且旧版本无记录价值。
- 文件名无意义且无项目关联，需要进一步确认。

禁止直接列为可删除：

- 项目文档。
- BIM/GIS 数据。
- 模型文件。
- 数据库文件。
- 历史项目源代码。
- 需求、汇报、原型、设计稿。
- 不确定用途的大压缩包。

## 整理流程

每轮整理遵循 5 步：

1. 选择一个目录。
2. 浅层扫描并生成索引。
3. 按本规则打分类别和优先级。
4. 生成整理建议清单。
5. 人工确认后，才进入移动/归档/删除计划。
