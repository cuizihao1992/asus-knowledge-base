# SUTPC 大屏地图交互摘要

生成日期：2026-06-21

范围：基于已有 SUTPC 需求摘要、GIS 数据需求详表和 BIM 集成流程摘要，提炼设施全生命周期监管大屏的地图交互逻辑。本文不读取、移动、删除、上传原始办公文档。

## 一句话概括

SUTPC 大屏地图不是静态展示页，而是一个“全市空间底板 + 项目/设施/模型/监测/视频入口”的交互中枢。它把规划、建设、养护、设施库、BIM 模型、监测预警和现场视频统一挂在地图对象上。

## 交互对象分层

| 层级 | 对象 | 交互意义 |
| --- | --- | --- |
| 底图层 | 全市路网、行政区划、水系、绿地、建筑普查、地名地址 | 提供空间定位和统计范围 |
| 交通设施层 | 道路、桥梁、隧道、边坡、场站、匝道、涵洞、交通标志、标线 | 设施库和管养业务入口 |
| 项目层 | 规划项目、在建项目、已建项目、重点项目、合同段 | 项目监管、进度、投资和资料入口 |
| BIM/三维层 | 设计 BIM、施工 BIM、竣工模型、3D Tiles、倾斜摄影、点云 | 模型查看、阶段切换、构件属性查询 |
| 监测视频层 | 监测点、传感器、视频、AI 识别、TSP、历史航拍 | 工程监管、预警和现场复核 |
| 周边专题层 | 政府单位、医院、学校、公交、轨道、枢纽、POI | 周边分析、应急和规划辅助 |

## 大屏页面结构

| 页面/模块 | 地图表达 | 点击后展示 | 关键联动 |
| --- | --- | --- | --- |
| 总览 | 全市路网图；规划/在建/已建项目分布；重点设施和预警点 | 项目概览、设施概览、预警详情 | 地图对象 -> 项目/设施详情 -> BIM/视频入口 |
| 重点项目 | 项目点线面、合同段、施工范围 | 项目信息、现场视频列表、BIM 模型跳转 | 项目 -> 合同段 -> 模型/视频/进度 |
| 重点设施 | 桥梁、隧道、边坡、场站等设施对象 | 设施信息、养护项目、建设项目、规划项目 | 设施 -> 历史项目 -> 养护/监测/模型 |
| 建设项目 | GIS 分布、工程范围、进度状态 | 项目基本信息、投资进度、进度偏差、任务状态 | 项目 -> BIM 构件 -> 计划/实际进度 |
| BIM 模型 | 初设、施工图、施工、竣工阶段模型 | 构件编码、材质、面积、体积、设计资料 | 模型阶段切换 -> 构件属性 -> 业务数据 |
| 三维管控 | 三维场景、施工区域、工程重难点 | 实时进度、工程变更、施工说明 | 时间/状态 -> 模型着色 -> 进度推演 |
| 施工监测 | 监测点、视频点、违规识别点、历史航拍 | 实时视频、AI 识别记录、TSP 数据、预警信息 | 监测点 -> 视频/指标 -> 预警处置 |

## 典型交互链路

### 1. 项目到 BIM 模型

```text
点击地图项目
  -> 打开项目基本信息
  -> 查看合同段、阶段、投资、进度
  -> 点击 BIM 模型入口
  -> 选择初步设计 / 施工图 / 施工 / 竣工阶段模型
  -> 查看构件属性和关联资料
```

关键字段：项目编号、项目名称、合同段、阶段、计划时间、实际时间、投资额、模型 geokey、模型 URL。

### 2. 设施到管养和监测

```text
点击桥梁 / 隧道 / 边坡 / 场站
  -> 打开设施基础信息
  -> 查看养护项目、建设项目、规划项目
  -> 查看监测点和预警状态
  -> 跳转视频或 BIM 模型
```

关键字段：设施编码、设施类型、所属道路、位置、技术状况等级、监测状态、养护项目编号。

### 3. BIM 构件到业务属性

```text
点击 BIM 构件
  -> 查询构件编码和基础属性
  -> 查看结构材质、面积、体积
  -> 关联投资进度、任务状态、设计文件
  -> 根据进度偏差做状态着色
```

关键字段：构件编码、Family、Type、结构材质、面积、体积、计划开始/完成、实际开始/完成、偏差值。

### 4. 监测点到视频和预警

```text
点击监测点 / 视频点
  -> 查看监测点基础信息
  -> 打开实时视频或历史视频
  -> 查看 AI 违规识别记录
  -> 查看预警等级和处置状态
```

关键字段：监测点编号、设备类型、视频编码、接入状态、预警等级、识别类型、处置状态。

## 前端设计要点

| 设计点 | 建议 |
| --- | --- |
| 图层控制 | 按底图、设施、项目、BIM、监测、视频、POI 分组，可单独开关 |
| 对象点击 | 每类对象点击后统一进入右侧属性面板，而不是弹出大量零散窗口 |
| 模型跳转 | 项目和设施详情中提供 BIM/三维入口，支持阶段切换 |
| 状态表达 | 项目进度、预警等级、设施技术状况用颜色或符号表达，需有图例 |
| 视频联动 | 视频点位点击后展示实时视频、AI 识别记录和关联项目 |
| 时间维度 | 建设进度和模型阶段应支持时间轴或阶段切换 |
| 大屏适配 | 需求中出现 8540mm x 2401mm 大屏，信息密度要高，避免普通网页式长页面 |

| 接口 | 必要字段 |

| 监测点列表 | `monitor_id`、`asset_id`、`type`、`geometry`、`status`、`warning_level`、`video_refs` |
| 视频列表 | `video_id`、`name`、`geometry`、`stream_url`、`project_id`、`asset_id`、`access_status` |

## 风险和缺口

| 风险 | 说明 | 后续动作 |
| --- | --- | --- |
| 坐标系不统一 | BIM、3D Tiles、矢量图层和视频点位可能不在同一空间参考 | 建立坐标系检查清单 |
| 模型服务地址缺失 | 本地只有原始模型或压缩包，没有发布后的 URL/geokey | 建立模型发布字段模板 |
| 视频权限和流地址 | 视频通常涉及权限、专网和流媒体协议 | 只记录元数据，不保存敏感地址 |
| 数据更新责任不清 | 项目、设施、监测、视频来自不同系统 | 为每类图层记录来源系统和责任单位 |
| 大屏信息过载 | 大屏图层多，容易变成杂乱堆叠 | 用场景模式控制图层组合 |
