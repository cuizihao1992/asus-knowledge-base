## 判断目的

这一步不是做视觉预览，而是判断这些本地模型是否具备进入 SUTPC BIM/GIS 服务链的基础条件：

```text
FBX 原始模型
  -> BIM 支撑平台或转换工具解析
  -> glTF / 3D Tiles / S3M / B3DM 等服务格式
  -> URL + geokey 或服务地址发布
  -> T-GIS / Cesium / 大屏 / 业务系统加载
```

## 只读元数据发现

| 检查项 | 发现 | 判断 |
| --- | --- | --- |
| FBX 格式 | 可检索到 `FBXVersion`、`FBX SDK/FBX Plugins version 2019.2` | 标准 FBX 文件，具备常规解析基础 |
| 来源软件 | 可检索到 `Autodesk Revit 2020`、`Autodesk Revit Export` | 来自 Revit，适合 BIM 支撑平台或 Revit/FBX 工具链处理 |
| 版本信息 | 可检索到 `FBX Version: 7.5.0` | 格式版本明确 |
| 单位信息 | 可检索到 `UnitScaleFactor` | 具备单位元数据，但数值和场景比例仍需打开确认 |
| 材质层 | 可检索到 `LayerElementMaterial`、材质 schema | 模型中存在材质/材料相关信息 |
| 构件属性 | 可检索到 `ObjectMetaData`、体积、面积、结构材质、Level、Family Name、Type Name 等字段 | 具备构件属性沉淀价值 |
| 纹理/材质资源 | 边坡和隧道文件中可见 `TextureMap2dSchema`、`UnifiedBitmapSchema` 等 schema | 可能含贴图/材质资源或引用，需打开验证是否缺外部文件 |

## 发布字段模板

后续每个 BIM 模型建议补充以下字段：

| 字段 | 说明 |
| --- | --- |
| 原始路径 | 本地文件路径 |
| 原始哈希 | SHA256，用于版本确认 |
| 来源软件 | 如 Revit 2020 |
| 原始格式 | 如 FBX |
| 候选发布格式 | glTF、3D Tiles、S3M、B3DM |
| 对象类型 | 桥梁、边坡、隧道、道路、管线等 |
| 设施编码 | 如 `LGBP0026`，没有则待补 |
| 坐标系 | 是否有地理坐标或需要手动配准 |
| 单位/比例 | 是否正常 |
| 材质 | 是否存在材质 |
| 贴图依赖 | 是否依赖外部贴图文件 |
| 构件层级 | 是否能展开构件树 |
| 构件属性 | 是否含体积、面积、材质、Level 等字段 |
| 业务关联 | 可关联项目、合同段、设施、视频、监测、养护 |
| 预览状态 | 未验证、可打开、打开失败、缺贴图、比例异常 |
| 转换状态 | 未转换、已转 glTF、已转 3D Tiles、转换失败 |
| 发布状态 | 未发布、已发布、已入 T-GIS、已入大屏 |
| 发布 URL | 如后续有 BIM 平台或 T-GIS 服务地址再填写 |
| geokey | 如后续由 BIM 平台生成再填写 |

5. 如果要进入 SUTPC/T-GIS 语境，再记录发布 URL、geokey、坐标配准和场景审批状态。
