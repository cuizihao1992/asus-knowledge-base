包装顺序

1. zhc-companion

  • 抽取“本地扫描规则”和“忽略目录规则”。
  • 对照当前 scripts/build-site.js，设计未来扫描脚本。
  • 不直接迁移代码，先迁移设计。

2. mini-transformer

  • 整理一页“从零理解 Transformer”的项目说明。
  • 保留训练命令、生成命令、API 服务命令。
  • 适合对外展示学习能力。

3. quant-system

  • 补数据流和模块图。
  • 记录 SQLite 表、回测输入输出、策略接口。
  • 不包装成投资建议。

4. pi-mono

  • 先读 docs-cn。
  • 后续按模块拆：CLI、runtime、tool、provider、TUI。
