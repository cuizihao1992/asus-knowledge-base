迁移原则

1. 只移动仓库内 Markdown 文档，不移动 D/E 盘原始资料。
2. 每批迁移后运行 node scripts/build-site.js。
3. README、状态看板和构建脚本分类规则要同步更新。
4. 自动生成文件和人工沉淀文件要分开，避免后续脚本覆盖人工文档。
5. 旧路径不作为长期入口；以 Git 历史追踪迁移过程。
