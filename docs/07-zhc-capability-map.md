## 可复用模式

### 1. 本地路径安全

来源：`zhc-companion`

关键思路：

- 指定工作区根目录。
- 所有路径解析后必须仍在工作区内。
- 忽略 `.git`、`node_modules`、缓存和构建产物。

可用于当前知识库：

- 后续如果做本地扫描脚本，应固定允许扫描的根目录。
- 不应让页面任意读取磁盘。
- 所有路径只记录索引，不上传原始大文件。

### 2. 项目卡片

来源：当前已整理的三个项目摘要。

统一字段：

| 字段 | 说明 |
| --- | --- |
| 项目名 | 本地目录名或真实项目名 |
| 路径 | 本地绝对路径 |
| 定位 | 项目解决的问题 |
| 技术栈 | 语言、框架、数据存储、服务方式 |
| 启动方式 | 最小可运行命令 |
| 核心模块 | 代码入口、文档入口、数据入口 |
| 可复用能力 | 对当前知识库有价值的设计 |
| 风险边界 | 不能直接迁移或公开的内容 |
| 后续动作 | 下一步整理或开发建议 |

### 3. 闭环优先

来源：`mini-transformer` 和 `quant-system`

一个项目是否值得优先整理，优先看它是否有闭环：

```text
输入 -> 处理 -> 结果 -> 保存/展示 -> 下一步动作
```

当前例子：

- `mini-transformer`：文本 -> 训练 -> checkpoint -> 生成 -> API 服务。
- `quant-system`：行情 -> SQLite -> 回测 -> 结果 -> 信号监控。
- 当前知识库：本地资料 -> 索引 -> Markdown -> HTML -> Pages 发布。
