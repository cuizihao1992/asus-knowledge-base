# 整理规范

## 基本原则

知识库先做索引，再做摘要。不要一次性把所有文件复制进仓库。

优先整理这些内容：

- 能反复查阅的技术资料。
- 项目需求、方案、汇报和关键交付文档。
- README、运行说明、环境配置。
- 自己写过或改过的代码项目。
- 已经确定有价值的 PDF、PPT、Word、Excel。

暂不优先整理这些内容：

- 安装包、运行库、缓存目录。
- 未确认内容的大压缩包。
- 系统目录和软件目录。
- 纯媒体文件，除非它和项目资料直接相关。

## 建议字段

每条资料索引建议包含：

| 字段 | 说明 |
| --- | --- |
| 标题 | 人能看懂的名称 |
| 模块 | 所属知识库模块 |
| 本地路径 | 原始文件或目录路径 |
| 类型 | PDF、Word、代码、图片、数据包、安装包等 |
| 来源 | 下载、项目、学习、个人整理等 |
| 状态 | 待整理、已索引、已摘要、已归档、可删除候选 |
| 重要程度 | 高、中、低 |
| 标签 | 便于搜索 |
| 摘要 | 一句话说明价值 |
| 后续动作 | 阅读、迁移、删除、补摘要、提炼随笔 |

## 状态定义

| 状态 | 含义 |
| --- | --- |
| 待整理 | 只知道路径，还没有判断价值 |
| 已索引 | 已记录路径、类型和模块 |
| 已摘要 | 已写出主要内容和用途 |
| 已归档 | 已放到明确模块，暂时无需处理 |
| 可删除候选 | 可能无价值、重复或过期，但删除前需要人工确认 |

## 命名建议

Markdown 文档建议使用：

```text
YYYY-MM-DD-主题.md
```

资料索引建议使用：

```text
模块-来源-主题
```

示例：

```text
GIS-Cesium-1.91示例入口
BIM-SUTPC-全生命周期需求文档
代码-mini-transformer-项目说明
学习-JavaScript深度学习-课程资料
```

## 第一阶段工作清单

1. 建立模块目录和说明文档。
2. 对 D、E 盘根目录和二级目录做浅层索引。
3. 先整理 `D:\zhc` 下近期代码项目。
4. 再整理 `E:\SUTPC` 和 `E:\Data` 下项目文档。
5. 最后处理 `E:\BaiduNetdiskDownload`、`E:\Download` 等下载池。

## 不建议的做法

- 不建议把整个 D/E 盘复制到仓库。
- 不建议直接提交大文件、压缩包、图片集或安装包。
- 不建议一开始就深度递归读取所有内容。
- 不建议在未确认用途前删除文件。
