# 个人知识体系路线图

生成日期：2026-06-20

范围：把当前知识库已经覆盖的 BIM/GIS、Cesium/WebGL、遥感测绘、AI/Transformer、量化系统、本地知识库工具串成一张个人能力路线图。

## 总目标

把本地资料和代码项目沉淀成一个可复用的个人能力体系：

```text
本地资料整理能力
  -> BIM/GIS/三维数据理解
  -> Web 三维展示和 Cesium 能力
  -> 遥感测绘基础
  -> AI/Transformer 学习能力
  -> 本地工具和自动化能力
  -> 可展示项目案例
```

## 优先级建议

| 优先级 | 方向 | 原因 |
| --- | --- | --- |
| P0 | 本地知识库工具 | 这是所有资料持续整理的底座 |
| P1 | SUTPC 项目经验 | 已有资料最多，最适合形成案例 |
| P1 | BIM/GIS/三维数据 | 能连接项目、数据资产、Cesium 和模型 |
| P2 | Cesium/WebGL | 支撑三维展示能力 |
| P2 | AI/Transformer | 保持为第二能力曲线 |
| P3 | 量化系统 | 有闭环，但和当前资料整理主线距离稍远 |
| P3 | 遥感测绘 | 有长期价值，适合慢慢精读 |

## 当前结论

继续堆文件清单的收益会下降。下一阶段更应该把知识库变成“能力地图”：每篇文档都回答它服务哪条主线、能证明什么能力、下一步能产出什么项目或案例。
