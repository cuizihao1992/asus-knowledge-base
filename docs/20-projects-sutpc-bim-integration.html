SUTPC BIM 数据集成流程专题摘要

生成日期：2026-06-20

一句话概括

SUTPC 全生命周期监管系统里的 BIM 不是单独的模型查看器，而是通过 BIM 支撑平台把设计、施工、运维阶段的模型文件解析成可发布、可检索、可叠加、可被 T-GIS 和业务系统调用的数据服务，再和项目、设施、进度、视频、监测、养护等业务数据建立关联。

平台关系

平台角色对全生命周期系统的能力
大数据平台数据归集和治理中心提供规划、前期、建设、养护等数据写入和读取服务；接收 BIM 平台推送的结构化业务数据
交通业务支撑平台业务能力组件提供算法请求、报表工具等能力
BIM 支撑平台BIM 数据接入、解析、转换、发布和场景编排中心提供 BIM 模型导入解析、可视化、数据查询、轻量化、场景编辑、服务发布
T-GIS 支撑平台GIS 空间底板和交通时空信息服务平台提供 GIS 数据导入解析、可视化、数据查询，并加载 BIM 平台发布的模型服务
智慧建设/智慧管养等既有平台存量业务和资产数据来源提供既有 BIM、点云、倾斜摄影、智能设施和业务属性数据

关键判断：BIM 支撑平台和 T-GIS 支撑平台共同构成公共服务层，前端或业务模块按需调用模型、图层、属性、场景和业务数据。

核心数据流

新增 BIM / GIS / 点云 / 倾斜摄影 / 影像等数据
  -> 上传到 BIM 支撑平台或通过服务接入
  -> BIM 平台解析、拆分、分类、转换、轻量化
  -> 生成标准三维服务和访问接口
  -> 场景编辑、配准、编排
  -> 发起场景发布审批
  -> 审批通过后发布服务
  -> T-GIS 通过 URL + geokey 加载协同展示
  -> 业务系统通过接口查询模型、构件、空间和业务属性

既有平台数据流：

智慧建设平台 / 智慧管养平台 / 既有业务平台
  -> 接口接入或文件同步
  -> BIM 平台整理转存
  -> 发布数据访问接口
  -> T-GIS 和业务系统调用

BIM 数据上传与发布

步骤动作产物
文件上传通过 BIM 前端上传模型文件到 BIM 服务器原始模型文件进入平台
静默处理BIM 服务自动对模型进行拆分、分类、解析入库模型元数据、文件索引、构件属性、空间属性
数据发布在 BIM 前端点击发布，触发模型数据发布流程可访问的三维服务
状态查看通过模型状态判断发布流程是否完成发布状态
分享访问已发布模型通过分享功能提供 URL可供 T-GIS 或前端访问的服务地址

需要保留的关键约束：

  • 模型坐标系要完整传递。
  • 材质、颜色等外观信息要完整传递。
  • 非几何属性要完整传递。
  • 发布后的模型要有唯一标识编码，方便与智慧建设平台等业务数据关联。

T-GIS 对接方式

BIM 平台对已完成发布的 S3M 模型，通过 URL + geokey 的方式访问。geokey 是模型唯一标识 ID，数据访问接口支持 HTTP 协议。

T-GIS 的调用逻辑：

步骤说明
获取模型列表T-GIS 调用 BIM 平台模型列表接口
读取 geokey模型信息接口返回每个模型的唯一标识
拼接访问地址T-GIS 按规则自行拼接 URL + geokey
前端加载T-GIS 前端通过 URL 访问 BIM 平台数据
协同展示在 GIS 三维场景中叠加 BIM 模型

这说明 BIM 与 GIS 的集成边界比较清晰：BIM 平台负责模型处理和服务发布，T-GIS 负责空间底板、场景融合和前端协同展示。

场景编辑与审批

流程说明
新建场景在三维场景管理菜单创建场景
编辑场景进入场景编辑页面
导入模型选择需要导入场景的模型数据
编排场景对场景数据进行自由编排和设置
发起发布场景挂接好数据后申请发布
审批流转经各级审批人审批
发布成功审批通过后场景正式发布

这个流程说明模型发布不是简单文件上线，而是带有场景编排和审批控制。对监管平台来说，这能保证领导驾驶舱、业务系统和公共服务层看到的是经过确认的空间场景。

对这个项目最有价值的经验是：BIM 模型从设计软件到 Web 三维场景，中间至少经过几何转换和渲染处理两个阶段。模型能不能在浏览器里流畅展示，主要取决于这两个阶段的处理质量。

可复用经验

1. BIM 平台的价值不在“打开模型”，而在把模型转换成统一、可发布、可检索、可关联的空间数据服务。
2. 大文件和属性数据必须分层：模型、切片、缓存放文件系统；元数据、索引、构件属性、场景编排放数据库。
3. URL + geokey 是 BIM 与 T-GIS 解耦的关键方式，T-GIS 不需要直接处理原始模型文件。
4. 场景发布要有审批和版本概念，否则大屏和业务系统可能加载到未确认的模型状态。
5. 进度、成本、合同、设施、养护、监测等业务信息应该通过唯一编码关联到 BIM 构件或模型对象。
6. 轻量化不是可选优化，而是 Web 三维场景能否实际使用的前置条件。

当前结论

SUTPC BIM 集成方案最值得沉淀的不是某个具体软件，而是一套数据工程模式：

  • 接入多源异构模型；
  • 转换成统一服务；
  • 通过唯一标识和业务数据关联；
  • 通过 T-GIS 进入空间场景；
  • 通过审批发布给业务系统和大屏使用；
  • 通过轻量化保证 Web 端可用。
