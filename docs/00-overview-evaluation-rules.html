资料价值评估与整理规则

生成日期：2026-06-19

目标：在真正整理 D/E 盘资料前，先统一“怎么评价价值、怎么分优先级、怎么处理文件”。本规则只用于生成整理建议，不代表自动移动或删除文件。

总原则

1. 不自动删除任何文件。
2. 不自动移动任何原始资料。
3. 不自动解压大压缩包。
4. 不上传模型、视频、数据库、安装包和原始项目资料。
5. 任何删除、移动、压缩、归档操作，都必须先生成清单，再由人工确认。
6. 默认操作是“索引”和“摘要”，不是“清理磁盘”。

资料分类

分类含义处理方式
核心保留明确有项目、经验、领域或复用价值建立索引，必要时做摘要
优先摘要文档或项目内容值得沉淀成知识生成 Markdown 摘要
资产索引模型、视频、GIS 数据、大文件只记录路径、体量、格式和关联关系
归档保留有历史价值，但短期不再处理记录后归档，不深入分析
待确认名称、来源或用途不清楚暂不处理，列入确认清单
可删除候选可能重复、过期、缓存或安装包只列为候选，不自动删除
禁止处理系统目录、权限敏感目录、运行环境关键目录不扫描或只浅层记录

优先级标准示例

可删除候选判定

注意：可删除候选不等于删除。

满足以下条件之一，可以列入“可删除候选”：

  • 明确是缓存目录。
  • 明确是构建产物。
  • 明确是重复安装包。
  • 明确是临时下载文件。
  • 已有更完整版本，且旧版本无记录价值。
  • 文件名无意义且无项目关联，需要进一步确认。

禁止直接列为可删除：

  • 项目文档。
  • BIM/GIS 数据。
  • 模型文件。
  • 数据库文件。
  • 历史项目源代码。
  • 需求、汇报、原型、设计稿。
  • 不确定用途的大压缩包。

整理流程

每轮整理遵循 5 步：

1. 选择一个目录。
2. 浅层扫描并生成索引。
3. 按本规则打分类别和优先级。
4. 生成整理建议清单。
5. 人工确认后，才进入移动/归档/删除计划。
